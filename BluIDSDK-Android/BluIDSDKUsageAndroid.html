BluID SDK Documentation

Introduction

        Embedding BluID SDK in your app, allows you to make use of a variety of BluB0X security features. BluID SDK eliminates carrying of physical ID/credentials to unlock the door in multiple facilities. BluID SDK, makes use of the phone's bluetooth adapter, for discovering nearby BluPOINT devices. These BluPOINT devices are attached to the door, turnstile etc, wherever there is an access restriction needed. If an authorized user has access to the door, then BluID SDK will authenticate the BluPOINT device with credentials to unlock the door.

 

Embedding BluID SDK in your application, will enable BluID SDK to make use phone’s of bluetooth adapter in the background, to find nearby BluPOINT devices. Thus enabling users of your application to seamlessly authenticate BluPOINT devices to open the access restricted door.

Users of your application will be able to authenticate BluPOINT devices, once their BluID cards are synchronized by activating BluID SDK API’s/function, which will take care of synchronizing BluID cards from BluSKY cloud server. In order to synchronize the BluID Cards, Users need to login once with their BluB0X credentials and BluID SDK will never ask for credentials for login once the user is successfully logged in.

  • Android SDK Build-tools 31

Setup Instructions

Create a BluB0X Occupancy account

Please contact BluB0X support for getting an BluB0X Occupance Manager account.

Create a General User Account

  • Login into https://blusky.blub0x.com with BluB0X Occupancy Manager account credentials
  • Click on Add User and enter user information and once added, the user will automatically get an email with login credentials.
  • Further if needed BluB0X Occupancy Manager can assign credential(s) to the user(s)

Initialization

  1. First step is to add “BluIDSDK.aar” as project dependency, and the SDK file will be provided by the BluBOX team. Steps to add :
  • Move the BluIDSDK.aar file into a folder from where it is accessible to AndroidStudio, e.g., in “app/BluIDSDK/BluIDSDK.aar”.
  • Add the following line of code into your  app’s gradle file’s dependencies section:

implementation files('BluIDSDK/BluIDSDK-release.aar')

  • Note: The path for the .aar file may be different in your case. Use that path.

  1. Next step is to add the required permissions into the app’s Manifest file. We’ll need permissions for Bluetooth, location, writing to storage, internet, and foreground service. To add the permissions, add following lines of code into your app’s Manifest file:

<uses-permission android:name="android.permission.BLUETOOTH" android:required="true"/>

<uses-permission android:name="android.permission.BLUETOOTH_ADMIN" android:required="true"/>

<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" android:required="true"/>

<uses-feature android:name="android.hardware.bluetooth_le" android:required="true"/>

<uses-permission android:name="android.permission.INTERNET" android:required="true"/>

<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" android:required="true"/>

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" android:required="true"/>

<uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>

<uses-permission android:name="android.permission.ACCESS_BACKGROUND_LOCATION"/>

  1. After adding the required permissions in the Manifest file, you need to request the permissions from the user. To do that, add the following global variables Activity

val appPermissions = arrayListOf<String>(

   Manifest.permission.BLUETOOTH,

   Manifest.permission.BLUETOOTH_ADMIN,

   Manifest.permission.ACCESS_FINE_LOCATION,

   Manifest.permission.INTERNET,

   Manifest.permission.READ_EXTERNAL_STORAGE,

   Manifest.permission.WRITE_EXTERNAL_STORAGE,

   Manifest.permission.FOREGROUND_SERVICE,

)

private val PERMISSION_REQUEST_CODE = 100

  1. After adding the global variables, you need to add the following lines of code into the onCreate function

 

if (ContextCompat.checkSelfPermission(

       this,

       Manifest.permission.ACCESS_FINE_LOCATION

   ) != PackageManager.PERMISSION_GRANTED ||

   ContextCompat.checkSelfPermission(

       this,

       Manifest.permission.BLUETOOTH

   ) != PackageManager.PERMISSION_GRANTED ||

   ContextCompat.checkSelfPermission(

       this,

       Manifest.permission.BLUETOOTH_ADMIN

   ) != PackageManager.PERMISSION_GRANTED ||

   ContextCompat.checkSelfPermission(

       this,

       Manifest.permission.WRITE_EXTERNAL_STORAGE

   ) != PackageManager.PERMISSION_GRANTED ||

   ContextCompat.checkSelfPermission(

       this,

       Manifest.permission.READ_EXTERNAL_STORAGE

   ) != PackageManager.PERMISSION_GRANTED

) {

   ActivityCompat.requestPermissions(

       this, arrayOf(

           Manifest.permission.ACCESS_FINE_LOCATION,

           Manifest.permission.BLUETOOTH,

           Manifest.permission.BLUETOOTH_ADMIN,

           Manifest.permission.WRITE_EXTERNAL_STORAGE,

           Manifest.permission.READ_EXTERNAL_STORAGE,

           Manifest.permission.FOREGROUND_SERVICE,

           Manifest.permission.ACCESS_BACKGROUND_LOCATION

       ), PERMISSION_REQUEST_CODE

   )

} else {

}

  1. Next step is to register the DeviceStateObserver to allow gesture based authentication  depending on access types (Refer link to view supported access type). To register the DeviceStateObserver, add the following line inside the application tag of your app’s Manifest file

<receiver android:name="com.blub0x.BluIDSDK.utils.DeviceStateObserver" ></receiver>

  1. Once DeviceStateObserver is added to your Manifest file, we need to create an instance of DeviceStateObserver and need to register the receiver in application’s Activity

import com.blub0x.BluIDSDK.utils.DeviceStateObserver

var deviceStateObserver = DeviceStateObserver()

this.registerReceiver(deviceStateObserver, filter)

  1. After the DeviceStateObserver has been registered, you need to register the BluIDSDK service with your app. To do that, add the following line inside the application tag of your app’s Manifest file

<service android:name="com.blub0x.BluIDSDK.utils.BLECentral" android:enabled="true" android:stopWithTask="true"/>

  1. After the service tag has been added in the Manifest file, we need to register it in the onCreate()

val serviceIntent = Intent(this, BLECentral::class.java)

startService(serviceIntent)

  1. You also need to stop the service when the application is closed. To do that, add the following lines of code in the onDestroy()

val serviceIntent = Intent(this, BLECentral::class.java)

stopService(serviceIntent)

  1. After all the set-up, the final step is to create an instance of BluIDSDK. To do that, first we have to import BluIDSDK class

import com.blub0x.BluIDSDK.BluIDSDK

  1. After importing the BluIDSDK class, we can instantiate the class like below

var m_BluIDSDK_Client = BluIDSDK(

   environment,

   activity,

   deviceStateObserver

)

  1. By default, BluIDSDK is set to a production environment, you can choose a different environment from multiple BluSKY environments list, pass your activity instance and deviceStateObserver that you created in step 6.

The initialization of your app is complete!

BluPOINT Device Discovery

Once the BluIDSDK is initialized, we can start discovering nearby BluPOINT Devices using the phone’s bluetooth adapter.

Discover BluPOINT devices

Before we start discovering BluPOINT devices, we need to make sure that, phone’s bluetooth is turned ON and we have the user's consent to use the bluetooth adapter. We can achieve this by using the following code block.

mBluIDSDK_Client.isBluetoothTurnedON()

mBluIDSDK_Client.isBluetoothAuthorized()

Then using discoverDevices API, we can now start discovering nearby BluPOINT devices,

mBluIDSDK_Client.let { sdkClient ->

   sdkClient.startDeviceDiscovery(

       ScanFilter(

           -80,

           1000

       )

   ) { bleDevices ->

       Unit

       for (device in bleDevices) {

           print(device)

       }

   }

}

Parameters:

  • filter: It defines the minimum signal strength of BluPOINT device and the time period at which the scan results should be shown while scanning.
  • onDeviceDiscovered : This callback is triggered at an interval defined in successiveCallbackFrequencyInMillis .
  • bleDevices: List of nearby BluPOINT devices.

Stop Device Discovery

We can stop ongoing device discovery, by using the following code block,

mBluIDSDK_Client.stopDeviceDiscovery()

User Login

BluID SDK incorporates BluB0X login to ensure a genuine user with BluB0X credentials is signed-in. Signing in with BluB0X credentials will enable users to make use of BluB0X security related features. Here is the code block which shows how login works.

mBluIDSDK_Client.let { sdkClient ->

   sdkClient.login(UserCredentials("username", "password"))

}

Parameters:

  • credentials: Credentials that need to be validated
  • onResponse : This callback is triggered if login succeeds.
  • error: if API fails this parameter is not set to nil. Refer link for possible error.
  • loginResponse: property of this parameter “expiresIn” contains session validity in seconds.

Refresh Login

After a successful login, we will receive session validity in response to the login API. We can extend the login session by refreshLogin API and here is a code block for reference

mBluIDSDK_Client.refreshLogin()

Response:

  • LoginResponse :
  • error: if API fails this parameter is not set to nil. Refer link for possible error.  
  • loginResponse: property of this parameter “expiresIn” contains session validity in seconds.

Gesture Based Authentication                        

Once the user is logged in, your app can start discovering nearby BluPOINT devices and interact with them. BluID SDK auto-detects for a user gesture like tap or twist and then authenticates the closest BluPOINT enabled door, using user credentials associated with PersonCard.

Sync Person Cards(online)

Person Cards with credentials are assigned to the user by BluB0X. PersonCard contains access to a particular set of BluPOINT devices, just like a key has access to locks.

Person Cards can be downloaded to the phone, using syncPersonCards API and here is the code block.

val personCards = mBluIDSDK_Client.syncPersonCards()

Response:

  • SyncCredentialsResponse:
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.
  • personCards: Returns the list of Person Cards.

List Person Cards(offline)

When we don't have connection with BluSKY server, we can still get a list of last synced PersonCards using getPersonCardsOffline API and here is the code block

val personCards = mBluIDSDK_Client.getPersonCardsOffline()

Response:

  • personCards: Returns the list of Person Cards.

Start Gesture based authentication

Once the Person Cards are synced from BluSKY server, we can start auto detection of gestures. Once the gesture is detected, BluID SDK will check for nearby BluPOINT enabled door’s. BluID SDK then checks for the closest gesture supported BluPOINT device and then authenticates using user credentials.

We can enable auto detection of gesture using the following code block

mBluIDSDK_Client.startGestureBasedAuthentication()

Stop Gesture based authentication

We can stop auto-detection of gesture using the following code block

mBluIDSDK_Client.stopGestureBasedAuthentication()


Manual Authentication

BluID SDK also supports manual selection of BluPOINT enabled doors and authenticating them.

User can select any BluPOINT enabled doors from the list of devices discovered and use the following code block to authenticate,

val deviceID = "00:0a:0b:0c:0d:0e"

mBluIDSDK_Client.transferCredential(deviceID)

Parameters:

  • deviceID: ID of unlocked BluPOINT device.
  • onResponse:
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Delete Person Cards

Users can also delete their Person Cards from their phone, using the deleteCredentials API and here is the code block on how to delete,

mBluIDSDK_Client.deleteAllPersonCardsFromUserDevice()

They can all delete all Person Cards using the following code block

val cardIDs = (arrayListOf("4", "6"))

mBluIDSDK_Client.deletePersonCardsFromUserDevice(cardIDs)

Parameter:

  • ​​cardIDs : List of card IDs to be deleted.

User Settings

BluID SDK maintains user preferences like which gestures they are interested in performing, whether they prefer haptic feedback and whether they want BluID SDK enabled app to run in background.

Get User Preferences

These user preferences are maintained by BluID SDK and can be listed using the following code block

   

mBluIDSDK_Client.getUserPreferences()

Response:

  • UserPreferences: Provides which gestures are enabled and disabled by the user.

If no setting is done by the user the default values are returned.

Save User Preferences

User can tweak their preferences like,

  • Enable/Disable any gesture from the list
  • Opt in/out for haptic feedback when the BluPOINT device is authenticated.
  • Decide whether the app should run in background or not by choosing the appropriate mode

Following is the code block where user can save their preferences,

val gestureSettings = UserPreferences(

   allowAccess = AllowAccessType.foreground, true, false, false, true, false,

   false, false, false

)

mBluIDSDK_Client.saveUserPreferences(gestureSettings)

Parameter:

  • setting : Allow users to choose access type(Foreground,Background or Always)  and enable any gesture based authentication.

Unlock BluPOINT Device

By default BluPOINT devices have restricted access to the general user, where access is limited to fetching details of BluPOINT device details and authenticating them using user credentials.

Only users with integrator access can unlock the restricted access to BluPOINT device and perform firmware update or tweak BluPOINT settings.

Please contact BluB0X support for getting an BluB0X Integrator access account.

Following is the code block will unlock BluPOINT device from restricted access

mBluIDSDK_Client.let { sdkClient ->

   sdkClient.unlockDeviceAccess(

       deviceID,

       onResponse = fun(error: BluIDSDKError?, unlockValidity: Long?) {

           print(error)

           print(unlockValidity)

       },

       disconnectCallback = { deviceID ->

           print("Device disconnected. ID: $deviceID")

       }

   )

}

Parameters:

  • deviceID: BluPOINT device Id.
  • onResponse : This callback is triggered once the device responds to the unlock request.
  • error:   if API fails this parameter is not set to nil. Refer link for possible error.
  • onDisconnect: This callback is triggered when the device gets disconnected.
  • disconnectResponse: BluPOINT device Id.

Locate/Identify BluPOINT Device

We can locate the BluPOINT device either by ringing a buzzer or a glowing LED on the BluPOINT device. Following code block shows the usage of identifyDevice API.

Glowing LED:

var bluPOINTLED = Device_LED(Device_LED_Color(0, 0xFF.toByte(), 0), 5)

var deviceID = "00:0a:0b:0c:0d:0e"

var identifyDeviceResponse = mBluIDSDK_Client.identifyDevice(bluPOINTLED, deviceID)

Parameters:

  • withLED : DeviceLED is the colour and time to which the LED of the unlocked device is set.       onResponse: This callback is triggered when the device responds to the glow LED request.
  • deviceID: Device ID of unlocked BluPOINT device which needs to be identified.

Response:

  • error:   if API fails this parameter is not set to nil. Refer link for possible error.

Ringing Buzzer:

val withBuzzerOnInSeconds = 5

val deviceID = "00:0a:0b:0c:0d:0e"

var identifyDeviceResponse =

   mBluIDSDK_Client.identifyDevice(deviceID, withBuzzerOnInSeconds)

Parameters:

  • withBuzzerONInSeconds : Time for which the buzzer is set ON.
  • deviceID: Device ID of unlocked BluPOINT device which needs to be identified.

Response:

  • error:   if API fails this parameter is not set to nil. Refer link for possible error.

Firmware Upgrade

Once the BluPOINT device is unlocked, Integrator user’s can perform firmware updates on BluPOINT devices. In order to update firmware on BluPOINT devices, we need to list and download supported firmware from the BluSKY server.

List Firmware

In order to update firmware we need to get available list of firmware, which can be obtained using the following code block

mBluIDSDK_Client.listAvailableFirmwareVersions()

Response:

  • firmwareList : List of available firmware versions in BluSKY server.

Download Firmware

Once the firmware list is obtained, users can download one or multiple firmware versions at once.

Firmware is downloaded on phones storage and is maintained by BluID SDK until the user explicitly deletes it from their phone. Following is the code block that shows how firmware downloaded

Single firmware version:

mBluIDSDK_Client.downloadFirmware("0:00:36")

Parameter:

  • version : Firmware version needs to download from BluSKY server.

Response:

  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

BluPOINT Firmware Update

Once the firmware is downloaded, the integrator user will be able choose the firmware version to update the BluPOINT device. Typically firmware updates take ~2mins to complete.

Here is the code block on how to update the firmware of BluPOINT device

val deviceID = "00:0a:0b:0c:0d:0e"

val version = "00.00.35"

mBluIDSDK_Client.updateSingleDeviceFirmware(

   deviceID,

   version,

   onProgress = fun(id: String, progress: Int) {

       print(id)

       print(progress)

   }

)

Parameters:

  • deviceID: Device ID of BluPOINT device whose firmware needs to be upgraded
  • version: Firmware version to be upgraded.
  • onProgress: This callback returns the percentage of firmware upgrade.
  • percent: Denotes the upgradation progress percentage.

Response:

  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Multi-Device Firmware Update

Integrator users can also perform firmware updates on multiple devices with a single API call. User can select multiple BluPOINT device from the list of discovered devices and choose which firmware version to update, using following code block we can perform multi device firmware update

val deviceList = arrayListOf<String>("00:0a:0b:0c:0d:0e", "10:1a:1b:1c:1d:1e")

val version = "00.00.35"

val updateResponse = mBluIDSDK_Client.updateMultipleDeviceFirmware(deviceList, version, progressHandler = { id, progress, updateError ->

   print(id)

   print(progress)

   print(updateError)

})

Parameters:

  • deviceList : List of IDs of BluPOINT devices.
  • version: The firmware version to which the devices will be upgraded.
  • progressHandler : This callback reports the firmware upgrade progress in percentage of particular deviceID.
  • progress: Denotes the upgrade progress percentage for the given deviceID.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Response:

  • updateResponse: Contains map of deviceID and their corresponding update status.

Delete Firmware

Integrator user can also delete the firmware from their phone using BluID SDK API and here is the following code block

val versions: ArrayList<String> = arrayListOf("0.00.35", "0.00.36")

var deleteFirmwareResponse = mBluIDSDK_Client.removeFirmwareFromUserDevice(versions)

Parameter:

  • versions: List of firmware versions to be deleted.

Response:

  • Contains map of firmware version and their corresponding status.

Tweaking BluPOINT Settings

Integrator user can tweak BluPOINT device settings like

  • Enable gesture support for BluPOINT devices and define the range in which user gesture’s can be processed and authenticated.
  • Tweak LED color to be illuminated when device is in-use or idle
  • Change name of BluPOINT device
  • Tweak BLE signal transmission strength

Following is the code block using which you can tweak BluPOINT Settings

Enable Gesture Support:

val setting =

   GenericBLESettings(-50, true)

val deviceID = "00:0a:0b:0c:0d:0e"

val updateDeviceSettingsResponse  = mBluIDSDK_Client.updateDeviceSettings(deviceID, GestureType.tap, setting)

Parameters:

  • deviceID: Device ID of BluPOINT device whose tap settings needs to be updated
  • gesture: gesture type to be updated. Refer link for list of gesture.
  • setting: Contains two parameters :

power:  range for the gesture to be detected

enable: flag to enable/disable the gesture.

Response:

  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Set LED Color:

val deviceID = "00:0a:0b:0c:0d:0e"

val deviceState = DeviceLEDState.idle

val ledColour = Device_LED_Color(0xFF.toByte(), 0xFF.toByte(), 0x80.toByte())

val setDeviceStateLEDColorResponse = mBluIDSDK_Client.setDeviceStateLEDColor(deviceID, deviceState, ledColour)

Parameters:

  • deviceID: Device ID of BluPOINT device whose idle/in-use led settings needs to be updated
  • deviceState: Denotes the state of the device i.e, idle or in_use.
  • ledColor: LED color for the given state.

Response:

  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Update Name:

val deviceID = "00:0a:0b:0c:0d:0e"

val newName = "Blu12"

mBluIDSDK_Client.updateDeviceName(deviceID, newName)

Parameters:

  • name: New device name that is to be used.
  • deviceID: Device ID of BluPOINT device whose name needs to be updated

Response:

  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Signal Transmission Strength:

val deviceID = "00:0a:0b:0c:0d:0e"

var updateDeviceBLETxPowerLevelResponse = mBluIDSDK_Client.updateDeviceBLETxPowerLevel(deviceID, 5)

Parameters:

  • deviceID: Device ID of BluPOINT device whose name needs to be updated
  • strength : BLE Tx power to be set for BluPOINT device.

Response:

  • error:  if API fails this parameter is not set to nil. Refer link for possible error.
  • strengthResponse: updated BLE Tx Power of BluPOINT device.

Lock BluPOINT Device

Integrator user who has unlocked the BluPOINT device can only lock the BluPOINT device using the following code block

val deviceID = "00:0a:0b:0c:0d:0e"

var disconnectDeviceResponse = mBluIDSDK_Client.lockDeviceAccess(deviceID)

Response:

  • BluIDSDKError: Response either error with BluIDSDKError not null or success response with BluIDSDKError set as null.

User Logout

If a user wishes to sign-out from their phone, BluID SDK provides an API to log out. This API will erase all the user specific data along with the all firmwares downloaded in the phone. Here is the code block which can be used for signing out the user from their phone.

mBluIDSDK_Client.logout()

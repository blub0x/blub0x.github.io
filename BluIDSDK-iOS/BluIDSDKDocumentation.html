BluID SDK Documentation

Introduction

        Embedding BluID SDK in your app, allows you to make use of a variety of BluB0X security features. BluID SDK eliminates carrying of physical ID/credentials to unlock the door in multiple facilities. BluID SDK, makes use of the phone's bluetooth adapter, for discovering nearby BluPOINT devices. These BluPOINT devices are attached to the door, turnstile etc, wherever there is an access restriction needed. If an authorized user has access to the door, then BluID SDK will authenticate the BluPOINT device with credentials to unlock the door.

 

Embedding BluID SDK in your application, will enable BluID SDK to make use of phone’s  bluetooth adapter in the background, to find nearby BluPOINT devices. Thus enabling users of your application to seamlessly authenticate BluPOINT devices to open the access restricted door.

Users of your application will be able to authenticate BluPOINT devices, once their Person cards are synchronized by activating BluID SDK API’s/function, which will take care of synchronizing Person cards from BluSKY server. In order to synchronize the Person cards, User need to login once with their BluB0X credentials.

Minimum Requirements

  • Xcode 12.5.1 or above
  • Xcode Deployment Target iOS 13 or above
  • Swift 5.3

Setup Instructions

Create a BluB0X Occupancy account

Please contact BluB0X support for getting an BluB0X Occupance Manager account.

Create a General User Account

  • Login into https://blusky.blub0x.com with BluB0X Occupancy Manager account credentials
  • Click on Add User and enter user information and once added, the user will automatically get an email with login credentials.
  • Further if needed BluB0X Occupancy Manager can assign credential(s) to the user(s)

Embedding BluIDSDK.framework in project

  • Copy BluIDSDK.framework file provided by BluB0X into you project’s directory
  • Import BluIDSDK.framework file into your project

Configure iOS privacy permissions

BluID SDK uses the phone's bluetooth adapter to discover nearby BluPOINT devices and interacts with them to perform operations.BluID SDK needs user consent for Bluetooth permissions. Following are the keys needs to be added in project’s Info.plist

  • NSBluetoothAlwaysUsageDescription
  • NSBluetoothPeripheralUsageDescription

Enable Background Modes

BluID SDK needs permission to run in the background, which allows your application to seamlessly authenticate BluPOINT devices without opening the application.

  • Open Project Settings
  • Navigate to Signing & Capabilities tab
  • Choose “+ Capabilities” option and select Background Modes
  • In Background Modes section, make sure to select option “Uses Bluetooth LE accessories” & “Background Processing”
  • In project info.plist, add the following key with bundle identifiers of BluID SDK & application.
  • BGTaskSchedulerPermittedIdentifiers
  • com.blub0x.BluIDSDK
  • <application_bundle_identifier>

BluID SDK Usage

Initialization

BluID SDK exposes all the API’s using the BluIDSDK instance. It is recommended to use a single instance of BluIDSDK throughout the application lifecycle.

First step is to import BluIDSDK in the application source file,

import BluIDSDK

Then initialize SDK using the following code block,

val m_bluIDSDKClient = BluIDSDK()

By default, BluIDSDK is set to production environment. You can also choose a different environment by using the following code block,

val m_bluIDSDKClient = BluIDSDK(env:BluIDSDK_Environment)

env: Environment parameter, is to choose from multiple BluSKY environments.

 

You can also find out what version of BluID SDK is being used by using the following code block

val sdkVersion:String = m_bluIDSDKClient.getSDKVersion()
print("BluIDSDK v\(sdkVersion)")

BluPOINT Device Discovery

Once the BluIDSDK is initialized, we can start discovering nearby BluPOINT Devices using the phone’s bluetooth adapter.

Discover BluPOINT devices

Before we start discovering BluPOINT devices, we need to make sure that, phone’s bluetooth is turned ON and we have the user's consent to use the bluetooth adapter. We can perform this check by using the following code block.

guard m_bluIDSDKClient.isBluetoothTurnedON() else {
        print("Bluetooth needs to be turned ON")
}
guard m_bluIDSDKClient.isBluetoothAuthorized() else {
        print("User consent is required to use Bluetooth")
}

Then using startDeviceDiscovery API, we can now start discovering nearby BluPOINT devices,

let filter = ScanFilter(signalStrength: -80, successiveCallbackFrequencyInMillis: 3000)

m_bluIDSDKClient.startDeviceDiscovery(filter: filter, onDeviceDiscovered: { (devices) in
   print(devices)
})

  • filter: It defines the minimum signal strength of BluPOINT device and the time period at which the scan results should be shown while scanning.
  • onDeviceDiscovered : This callback is triggered at an interval defined in successiveCallbackFrequencyInMillis .
  • devices: List of nearby BluPOINT devices.

Stop Device Discovery

We can stop ongoing device discovery, by using the following code block,

 m_bluIDSDKClient.stopDeviceDiscovery()

User Login

BluID SDK incorporates BluB0X login to ensure a genuine user with BluB0X credentials. Signing in with BluB0X credentials will enable users to make use of BluB0X security related features. Here is the code block which shows how login works.

m_bluIDSDKClient.login(credentials: UserCredentials(username:"username", password:"password"), onResponse: { (error, loginResponse) in

        if let _error = error {

           print(_error)

           return

        }
       print("session validity: \(loginResponse.expiresIn)")
})

  • onResponse : This callback is triggered if login succeeds.
  • error: if API fails this parameter is not set to nil. Refer link for possible error.
  • loginResponse: property of this parameter “expiresIn” contains session validity in seconds.

Refresh Login

After a successful login, we will receive session validity in response to the login API. We can extend the login session by refreshLogin API and here is a code block for reference

 m_bluIDSDKClient.refreshLogin(onResponse: { (error, loginResponse) in
       if let _error = error {

           print("login failed")

           print(_error)

           return

        }
       print("session validity: \(loginResponse.expiresIn)")
})

  • onResponse : This callback is triggered when the BluSKY server responds to the login request.
  • error: if API fails this parameter is not set to nil. Refer link for possible error.  
  • loginResponse: property of this parameter “expiresIn” contains session validity in seconds.

Gesture Based Authentication                        

Once the user is logged in, your app can start discovering nearby BluPOINT devices. BluID SDK auto-detects for a user gesture like tap or twist and then authenticates the closest BluPOINT enabled door, using user credentials associated with PersonCard.

Sync Person Cards(online)

Person Cards with credentials are assigned to the user by BluB0X support. PersonCard contains access to a particular set of BluPOINT devices, just like a key has access to locks.

Person Cards can be downloaded to the phone, using syncPersonCards API and here is the code block.

m_bluIDSDKClient.syncPersonCards(onComplete:{ (error, personCards)  in

        if let _error = error {

           print("Sync Person cards failed")

           print(_error)

           return

        }
       print("Total Person cards: \(personCards.count)")
})

  • onComplete : This callback is triggered when the BluSKY server responds to the syncPersonCars Request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.
  • personCards: Returns the list of Person Cards.

List Person Cards(offline)

When we don't have connection with BluSKY server, we can still get a list of last synced PersonCards using getPersonCardsOffline API and here is the code block

let personCards = m_bluIDSDKClient.getPersonCardsOffline()
   print(personCards)

  • personCards: Returns the list of Person Cards.

Start Gesture based authentication

Once the Person Cards are synced from BluSKY server, we can start auto detection of gestures. Once the gesture is detected, BluID SDK will check for the closest nearby gesture enabled BluPOINT door and authenticates using user credentials associated with PersonCard.

We can enable auto detection of gesture using the following code block

m_bluIDSDKClient.startGestureBasedAuthentication()

Stop Gesture based authentication

We can stop auto-detection of gesture using the following code block

m_bluIDSDKClient.stopGestureBasedAuthentication()

Manual Authentication

BluID SDK also supports manual selection of BluPOINT enabled doors and authenticating them.

User can select any BluPOINT enabled doors from the list of devices discovered and use the following code block to authenticate,

m_bluIDSDKClient.transferCredential(deviceID:"00:0a:0b:0c:0d:0e",
onResponse:{ (error)  in
       if let _error = error {

           print("Failed to transfer credentials")

           print(_error)

           return

        }
})

  • deviceID: ID of unlocked BluPOINT device.
  • onResponse:This callback is triggered when the BluPOINT device responds to a transfer credential request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Delete Person Cards

Users can also delete their Person Cards from their local storage, using the deleteCredentials API and here is the code block on how to delete,

m_bluIDSDKClient.deleteAllPersonCardsFromUserDevice()

They can all delete all Person Cards using the following code block

let cardIDs = ["4","6"]
m_bluIDSDKClient.deletePersonCardsFromUserDevice(cardIDs:cardIDs)

  • ​​cardIDs : List of card IDs to be deleted.

User Settings

BluID SDK maintains user preferences like which gestures they are interested in performing, whether they prefer haptic feedback and whether they want to allow application to run in the background.

Get User Preferences

User preferences are maintained by BluID SDK and can be listed using the following code block

   

let userPreferences = m_bluIDSDKClient.getUserPreference()
print(userPreferences)

Save User Preferences

User can tweak their preferences like,

  • Enable/Disable any gesture from the list
  • Opt in/out for haptic feedback when the BluPOINT device is authenticated.
  • Decide whether the app should run in background or not by choosing the appropriate mode

Following is the code block where user can save their preferences,

let userPreferences = UserPreferences(
allowAccess: AllowAccessType.foreground , enableVibrate : false, enableTap:true, enableTwist:true, enableAppSpecific:false, enableWave:false, enableRange: true, enableAI: false, enableAppleWatch: false, enableBluREMOTE:false, enhancedTap: true )
m_bluIDSDKClient.saveUserPreference(setting: userPreferences)

  • setting : Allow users to choose access type(Foreground, Background or Always)  and enable any gesture based authentication.

Unlock BluPOINT Device

By default BluPOINT devices have restricted access to the general user, where access is limited to fetching details of BluPOINT devices and authenticating them using user credentials.

Only users with integrator access can unlock the restricted access to BluPOINT device and perform firmware update or tweak BluPOINT settings.

Please contact BluB0X support for getting an BluB0X Integrator access account.

Following is the code block will unlock BluPOINT device from restricted access

m_bluIDSDKClient.unlockDeviceAccess(deviceID: "00:0a:0b:0c:0d:0e",
      onResponse:{ (error, successResponse) in
           if let _error = error {

               print("Unlock device failed")

               print(_error)

               return

            }
      },
      onDisconnect: { (deviceID) in
          print("device disconnected: \(deviceID)")
})

  • deviceID: BluPOINT device Id.
  • onResponse : This callback is triggered once the device responds to the unlock request.
  • error:   if API fails this parameter is not set to nil. Refer link for possible error.
  • onDisconnect: This callback is triggered when the device gets disconnected.
  • disconnectResponse: BluPOINT device Id.

Locate/Identify BluPOINT Device

We can locate the BluPOINT device either by ringing a buzzer or a glowing LED on the BluPOINT device. Following code block shows the usage of identifyDevice API.

Glowing LED:

let deviceLED = Device_LED(color: Device_LED_Color(red:0,green:0xff,blue:0),onTimeInSeconds: UInt8(5))

m_bluIDSDKClient.identifyDevice(withLED:deviceLED,onResponse:{ (error)  in
 if let _error = error {

     print("Failed to identify device")

     print(_error)

     return

  }
})

  • withLED : DeviceLED is the colour and time to which the LED of the unlocked device is set.       onResponse: This callback is triggered when the device responds to the glow LED request.
  • error:   if API fails this parameter is not set to nil. Refer link for possible error.

Ringing Buzzer:

let buzzerONInSeconds = UInt8(5)
        m_bluIDSDKClient.identifyDevice(withBuzzerONInSeconds:UInt8(buzzerONInSeconds),onResponse:{ (error)  in
   if let _error = error {

       print("failed to identify device")

       print(_error)

       return

    }
})

  • withBuzzerONInSeconds : Time for which the buzzer is set ON. 
  • onResponse: This callback is triggered when the device responds to the ring buzzer request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Firmware Upgrade

Once the BluPOINT device is unlocked, Integrator user’s can perform firmware updates on BluPOINT devices. In order to update firmware on BluPOINT device, we need to list and download supported firmware from the BluSKY server.

List Firmware

In order to update firmware we need to get available list of firmware, which can be obtained using the following code block

m_bluIDSDKClient.listAvailableFirmwareVersions(onResponse:{ (firmwareList) in
  print(firmwareList)
})

  • onResponse : This callback is triggered when the BluSKY server responds to the available firmware request.
  • firmwareList : List of available firmware versions in BluSKY server.

Download Firmware

Once the firmware list is obtained, users can download the firmware version using BluIDSDK API. Firmware is downloaded on phone’s storage and is maintained by BluID SDK until the user explicitly deletes it from their phone. Following is the code block that shows how to download firmware.

Single firmware version:

m_bluIDSDKClient.downloadFirmware(version:"0.00.36",onComplete:{ (error)  in
  if let _error = error {

      print("Download firmware failed")

      print(_error)

      return

   }
})

  • version : Firmware version needs to download from BluSKY server.
  • onComplete: This callback is triggered when the BluSKY server responds to the download firmware request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

BluPOINT Firmware Update

Integrator users can choose the downloaded firmware version to update the BluPOINT device. Typically firmware update takes ~2mins to complete.

Here is the code block on how to update the firmware of BluPOINT device

m_bluIDSDKClient.updateSingleDeviceFirmware(version:"0.00.36",
onUpdateComplete:{ (error, deviceConnectionState)  in

      if let _error = error {

               print("Firmware update failed")

               print(_error)

               return

      }
     print("Device \(deviceConnectionState)")
},
progressHandler : {(percent) in
    print("Update progress: \(percent)%)
})

  • version : Firmware version to be upgraded.
  • onUpdateComplete : This callback is triggered when the firmware upgrade is complete.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.
  • deviceConnectionState : ‘True’ if the device is connected and ‘False’ if the device is disconnected.
  • progressHandler : This callback returns the percentage of firmware upgrade.
  • percent: Denotes the upgradation progress percentage.

Multi-Device Firmware Update

Integrator users can also perform firmware updates on multiple devices with a single API call. User can select multiple BluPOINT device from the list of discovered devices and choose which firmware version to update, using following code block we can perform multi device firmware update

let deviceIDsList = ["00:0a:0b:0c:0d:0e","00:0a:0b:0c:0d:0f"]
let version = "0.00.36"
       
m_bluIDSDKClient.updateMultipleDeviceFirmware(deviceIDs:deviceIDsList,
firmwareVersion: version,
onUpdateComplete:{ (updateResponse)  in
   print(updateResponse)
},
progressHandler : {(deviceID, percent, error) in
   if let _error = error {

               print("Firmware update failed")

               print(_error)

               return

    }
   print("Update progress: \(percent)% of device: \(deviceID))
})

  • deviceIDs : List of IDs of BluPOINT devices.
  • firmwareVersion: The firmware version to which the devices will be upgraded.
  • onUpdateComplete : This callback is triggered when the upgrade process is complete.
  • updateResponse: Contains map of deviceID and their corresponding update status.
  • progressHandler : This callback reports the firmware upgrade progress in percentage of particular deviceID.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.
  • percent: Denotes the upgrade progress percentage for the given deviceID.

Delete Firmware

Integrator user can also delete firmware from their phone using BluID SDK API and here is the following code block

let versions = ["0.00.35", "0.00.36"]
let response = m_bluIDSDKClient.removeFirmwareFromUserDevice(versions:versions)
print(response)

  • versions: List of firmware versions to be deleted.
  • response: Contains map of firmware version and their corresponding status.

Tweaking BluPOINT Settings

Integrator user can tweak BluPOINT device settings like

  • Enable gesture support for BluPOINT devices and define the range in which user gesture’s can be processed and authenticated.
  • Tweak LED color to be illuminated when device is in-use or idle state.
  • Change name of BluPOINT device
  • Tweak BLE signal transmission strength

Following is the code block using which you can tweak BluPOINT Settings

Enable Gesture Support:

let tapSettings = GestureSetting(enabled: true, power: -50)
let gestureType = GestureType.tap
   
m_bluIDSDKClient.updateDeviceSettings(gesture: gestureType, setting: tapSettings, callback:{ (error)  in
  if let _error = error {

               print("Failed to update device settings")

               print(_error)

               return

    }
})

  • gesture: gesture type to be updated. Refer link for list of gesture.
  • setting: Contains two parameters :

power:  range for the gesture to be detected

enable: flag to enable/disable the gesture.

  • Callback : This callback is triggered when the device responds to updateDeviceSettings request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Set LED Color:

let ledColor = Device_LED_Color(red:0,green:0xff,blue:0)
   
m_bluIDSDKClient.setDeviceStateLEDColor(forDeviceState: deviceState,color:ledColor,onResponse:{ (error)  in
      if let _error = error {

               print("Failed to update LED color")

               print(_error)

               return

       }
})

  • forDeviceState: Denotes the state of the device i.e, idle or in_use.
  • color : LED color for the given state.
  • onResponse : This callback is triggered when the device responds to setDeviceStateLEDColor Request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Update Name:

let deviceName = "device1"
       
m_bluIDSDKClient.updateDeviceName(name: deviceName,onResponse:{ (error)  in
   if let _error = error {

               print("Failed to update device name")

               print(_error)

               return

    }
})

  • name: name to be updated.
  • onResponse : This callback is triggered when the device responds to the updateDeviceName request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.

Signal Transmission Strength:

let strength: UInt8 = 3
m_bluIDSDKClient.updateDeviceBLETxPowerLevel(strength: strength,
callback:{ (error,strengthResponse)  in
   if let _error = error {

               print("Failed to update device BLE Transmission power")

               print(_error)

               return

    }

    print("Strength is set to \(strengthResponse)")
})

  • strength : BLE Tx power to be set for BluPOINT device.
  • Callback : This callback is triggered when the device responds to updateDeviceBLETxPowerLevel Request.
  • error:  if API fails this parameter is not set to nil. Refer link for possible error.
  • strengthResponse: updated BLE Tx Power of BluPOINT device.

Lock BluPOINT Device

Integrator user who had unlocked the BluPOINT device, can only lock back the BluPOINT device using the following code block

m_bluIDSDKClient.lockDeviceAccess()

User Logout

If a user wishes to logout from their phone, BluID SDK provides an API, which will erase all the user specific data along with all firmware downloads from the phone. Here is the code block which can be used for signing out the user from their phone.

m_bluIDSDKClient.logout()
